# src

Окремі точки входу — кожен файл самодостатній міні-сервер для запуску через `tsx` або скрипти з `package.json`.

## Файли прикладів

- [`server.ts`](./server.ts) — мінімальний каркас: middleware, що пропускає запит далі (`next`), і один маршрут з відповіддю-даними; база для розуміння «що виконується до handler».

- [`server_start.ts`](./server_start.ts) — найпростіший старт: текстова відповідь з кореня та окремий «діагностичний» JSON-ендпоінт (стан сервера в широкому сенсі).

- [`server_verbs.ts`](./server_verbs.ts) — один URL-ресурс і різні HTTP-методи: як Express розводить `GET` / `POST` / `PATCH` / `PUT` / `DELETE` на одному шляху без прив’язки до БД.

- [`server_send_data.ts`](./server_send_data.ts) — невелике JSON API поверх масиву в пам’яті: робота з query, динамічними сегментами шляху, створенням і частковим оновленням запису — ідея «дані як заглушка, фокус на маршрутах і контракті відповіді».
