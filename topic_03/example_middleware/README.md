# example_middleware

Приклади на роль middleware у Express: що робить шар між запитом і фінальною відповіддю.

## Файли прикладів

- [`ex01.ts`](./ex01.ts) — один глобальний middleware (лог) і маршрут: як код «на вході» виконується перед handler.

- [`ex02.ts`](./ex02.ts) — кілька `app.use` підряд: наочно видно порядок проходження запиту по шарах до роута.

- [`ex03.ts`](./ex03.ts) — middleware лише на `POST`: перевірка обов’язкових полів тіла перед логікою створення запису.

- [`ex04.ts`](./ex04.ts) — ланцюг middleware на одному маршруті: лог → валідація → бізнес-умова (дублікат) → handler.

- [`ex05.ts`](./ex05.ts) — middleware додає на `req` службове поле (час), яке читає handler: ідея «збагачення запиту» для наступних кроків.
